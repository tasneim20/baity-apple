# لوحة تحكم الأدمن — منصة بيتي

## نظرة سريعة

لوحة التحكم تغطي كل شيء يحتاجه الأدمن لإدارة المنصة: مراجعة العقارات قبل نشرها، متابعة البلاغات، الاطلاع على المحادثات، مراقبة عمليات البيع والشراء، إدارة المستخدمين، واستيراد عقارات من CSV. الدخول عبر `admin@baity.com` أو اختصار `Ctrl + Shift + A` من أي صفحة.

## آلية مراجعة العقارات

كل عقار يُرسله مستخدم يُحفظ بحالة `pending` تحت المفتاح `pending_property_{id}` في KV Store، ولا يظهر في أي قائمة عامة حتى يُوافق عليه الأدمن. عند الموافقة يُنقل إلى `property_{id}` وتتغير حالته إلى `approved`. عند الرفض ينتقل إلى `rejected_property_{id}` مع حفظ سبب الرفض في حقل `rejectionReason`.

الأدمن يرى الطلبات المعلقة في `/admin/properties`، ويملك الإجراءات التالية على أي عقار:
- **موافقة** — نشر فوري
- **رفض** — مع سبب اختياري (8 أسباب جاهزة + حقل نص حر)
- **تعديل** — يُعدّل مباشرة دون إرجاعه للحالة pending
- **حذف** — يُزال من KV فوراً بأي حالة كانت (pending/approved/rejected)
- **تمييز** — toggle لحقل `featured` للإعلانات الممولة
- **تغيير حالة التوفر** — متاح / تم البيع

كل عملية من الأعلى تُسجَّل تلقائياً في KV تحت مفتاح `admin_log_{uuid}`.

## نظام البلاغات

المستخدمون يُبلّغون عن العقارات من صفحة التفاصيل. كل بلاغ يُخزَّن كـ `report_{uuid}` في KV ويحتوي على لقطة من بيانات العقار وقت الإبلاغ (العنوان، المحافظة، اسم المالك).

الأدمن يرى البلاغات في `/admin/reports` مع التبويبات: معلّق / تمت المراجعة / تم اتخاذ إجراء / مُهمل. الإجراءات المتاحة:
- `delete_property` — يحذف العقار من KV
- `hide_property` — يضع `status: "hidden"` على العقار
- `warn_user` — تسجيل تحذير بدون إجراء على العقار
- `dismiss` — إغلاق البلاغ بدون إجراء

Rate limiting على إرسال البلاغات: 10 بلاغات / ساعة لكل مستخدم.

## المحادثات وعمليات البيع

**AdminMessages:** يعرض كل المحادثات القائمة في المنصة بصرف النظر عن أصحابها. الأدمن يقدر يحذف رسالة معينة أو محادثة كاملة بدون أي قيد زمني (المستخدم العادي لديه 10 دقائق فقط). كل حذف أو تعديل يُسجَّل في Admin Logs.

**AdminTransactions:** نفس بيانات المحادثات لكن من منظور تجاري — يعرض اسم المستفسر والمالك ومعلومات العقار وعدد الرسائل، مع إمكانية تحديث حالة العملية:
- `pending` → `in_progress` → `completed` / `cancelled`

حالة العملية تُخزَّن منفصلة تحت `tx_status_{threadId}` حتى لا تُلوّث بيانات المحادثة الأصلية.

## استيراد CSV

صفحة `/admin/csv` تقبل ملف CSV يحتوي عقارات جاهزة للنشر. توزيع المحافظات يعمل بنظام Round-Robin على المحافظات الـ 12 إذا لم تُحدَّد المحافظة في الملف. بعد الاستيراد يمكن الانتقال مباشرة إلى `/admin/added-properties?source=csv` لمراجعة ما تم استيراده.

المحافظات الـ 12 مدعومة:
عمّان، الزرقاء، إربد، العقبة، المفرق، البلقاء، الكرك، مأدبا، جرش، عجلون، معان، الطفيلة.

مقارنة المحافظات تمر عبر `normalizeGov()` في `src/app/utils/governorateUtils.ts` لتوحيد صيغة الكتابة بين العربية والإنجليزية والتشكيل.

## إدارة المستخدمين

`/admin/users` يجلب المستخدمين من Supabase Auth عبر `admin.listUsers()` (حد 1000 مستخدم). لكل مستخدم يعرض: الاسم، البريد، رقم الهاتف، تاريخ التسجيل، آخر دخول، وعدد العقارات. عند حذف مستخدم تُحذف معه: عقاراته، مفضلته، محادثاته، وبلاغاته — كل هذا يحدث تسلسلياً داخل edge function.

ملاحظة: `messagesCount` يُعاد دائماً كـ `0` لأن تحميل محتوى الـ threads الكاملة للحساب يُسبّب timeout في edge function.

## سجل العمليات

كل ما يفعله الأدمن يُسجَّل عبر `writeAdminLog()` الموجودة في `index.tsx`. السجل يشمل:

| الحقل | القيمة |
|---|---|
| `admin_id` | UUID من Supabase Auth |
| `admin_email` | `admin@baity.com` |
| `action` | approve / reject / delete / message_edit / message_delete / report_action |
| `entity_type` | property / message / report |
| `entity_id` | ID الكيان المتأثر |
| `details` | كائن JSON بالتفاصيل (العنوان، السبب، إلخ) |
| `created_at` | ISO timestamp |

`/admin/logs` يجلب آخر 500 سجل مرتبة تنازلياً. الصفحة فيها فلترة بالنوع وبحث نصي في التفاصيل.

## الأمان

التحقق من الأدمن يتم في `requireAdmin()` عبر مقارنة `user.email === "admin@baity.com"`. لا يوجد نظام أدوار متعدد — الأدمن هو حساب واحد فقط. كل endpoints الأدمن تُعيد `403` فوراً لأي مستخدم آخر.

حساب الأدمن يُنشأ تلقائياً عند بدء الـ edge function إذا لم يكن موجوداً (`email_confirm: true` بدون بريد تأكيد).
